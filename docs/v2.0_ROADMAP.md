# SBTI+ 小程序下阶段路线图（学生增长版）

> 目标人群：初中及以上学生  
> 核心目标：积累流量、提升分享裂变、形成稳定引流与轻量变现闭环  
> 文档定位：承接当前已上线版本，为下一阶段开发、内容运营和商业化试水提供统一方向

---

## 1. 当前项目基线

### 1.1 已完成能力

- SBTI 星座人格测试主流程可用
- 室友匹配、CP 配对、每日测试、学习风格、星座运势入口已上线
- 已完成一轮关键 Bug 修复，核心答题与结果页稳定性提升
- 已补充微信内分享能力：
  - 结果页分享卡片
  - 页面自定义分享文案
  - 朋友圈分享
  - 海报生成与保存

### 1.2 当前短板

| 方向 | 当前问题 | 直接影响 |
|------|----------|----------|
| 内容供给 | 题量偏少，重复游玩价值不足 | 留存偏低 |
| 学生场景 | 初中/高中/大学场景还不够细 | 代入感不足 |
| 裂变机制 | 目前偏“能分享”，但缺少“想分享、愿转发”的触发点 | 自传播弱 |
| 数据沉淀 | 缺少用户画像、分享转化、活跃数据 | 难以优化 |
| 私域引流 | 缺少从测试结果到社群/公众号/企微的承接 | 流量流失 |
| 商业化 | 还没有轻量付费试探入口 | 无法验证变现模型 |

---

## 2. 下阶段总策略

下一阶段不建议一上来做重后端和重付费，而是围绕“**内容扩容 + 分享裂变 + 数据打底 + 轻量变现试水**”四条线推进。

### 2.1 阶段目标

1. **让用户更愿意测**：补充更贴近学生生活的测试内容
2. **让用户更愿意发**：把结果页做成适合朋友圈、宿舍群、同学群传播的社交内容
3. **让用户愿意回访**：增加周期性内容和记录体系
4. **让流量可沉淀**：为后续社群、公众号、私域、会员打基础
5. **让商业化有验证**：先试低门槛付费，再决定是否做重会员体系

### 2.2 产品关键词

`学生人格测试` / `社交破冰` / `宿舍关系` / `暗恋话题` / `考试情绪` / `轻分享` / `低门槛付费`

## 3. 用户与场景拆解

### 3.1 核心受众

| 人群 | 主要需求 | 适合内容 |
|------|----------|----------|
| 初中生 | 趣味测试、同学互动、星座话题 | 轻松、短平快、结果明确 |
| 高中生 | 学习压力、关系敏感、情绪表达 | 学习状态、暗恋、友情、考前状态 |
| 大学生 | 宿舍关系、恋爱、社交、未来规划 | 室友匹配、CP 配对、实习/专业/社团 |

### 3.2 高频传播场景

- 测出“很像我”的结果后发朋友圈
- 在宿舍群/班级群发起“你也来测”
- 情侣、暧昧对象、朋友之间做配对和 PK
- 考前、开学、分班、军训、情人节等节点带动集中传播

## 4. 功能规划（按优先级）

### 4.1 P0：增长优先，尽快上线

### A. 学生题库扩容

新增题库不追求一次做太多，先把最容易形成分享的话题补齐。

| 模块 | 建议新增 | 重点方向 |
|------|----------|----------|
| 性格测试扩展题 | +20 ~ +30 | 宿舍、上课、社交、考试、暗恋、群聊 |
| 每日测试题库 | +30 | 一题一结果，适合天天打开 |
| 学习状态测试 | +12 | 拖延、复习节奏、考前焦虑、课堂风格 |
| 社交关系测试 | +12 | 同桌、闺蜜、兄弟、宿舍、暧昧对象 |

**落地原则：**
- 单套题量控制在 8~15 题，减少流失
- 题目文案口语化，贴近学生表达
- 结果标题要“可截图、可转发、可调侃”

### B. 结果页裂变升级

在现有分享功能基础上继续强化：

1. 增加“更适合发圈”的结果标题  
2. 增加“好友也来测”的 CTA 文案  
3. 针对不同测试类型输出不同海报模板  
4. 结果页增加“重新测试 / 分享给朋友 / 发起配对”三连动线

**建议新增文案方向：**
- “你是宿舍里最容易被依赖的人”
- “你属于班里表面冷静、内心戏超多型”
- “你的暗恋脑浓度已超标”

### C. 活动化专题测试

围绕学生日历做专题入口：

| 节点 | 专题建议 |
|------|----------|
| 开学季 | 你的新学期人设、宿舍生存力测试 |
| 期中/期末 | 考前状态测试、崩溃指数测试 |
| 情人节/520/七夕 | 暧昧升温指数、CP 合拍测试 |
| 暑假 | 假期摆烂指数、社交恢复测试 |

这类内容传播效率通常高于常规人格题，适合做短周期拉新。

---

### 4.2 P1：增强留存与数据沉淀

### A. 用户记录体系

建议先做轻量版，不急于重账号体系：

- 本地历史记录页
- 最近测试结果回看
- 最近配对记录回看
- 最近分享海报缓存

第二步再升级为云端同步：

- 微信授权登录
- 历史测试云端保存
- 换设备可恢复

### B. 数据埋点体系

下阶段必须补上最基础的数据采集。

| 指标 | 用途 |
|------|------|
| 首页入口点击率 | 判断什么功能最吸引人 |
| 测试开始率 | 判断入口文案是否有效 |
| 测试完成率 | 判断题量和题目质量 |
| 分享触发率 | 判断结果是否具备传播性 |
| 海报保存率 | 判断分享图是否有效 |
| 次日回访率 | 判断内容是否有持续吸引力 |

### C. 内容后台雏形

先不做复杂 CMS，可先做最小可用内容管理：

- 题库 JSON 配置化
- 每日测试按日期投放
- 星座文案可替换
- 节日专题可快速上下线

---

### 4.3 P2：引流与商业化试水

### A. 私域引流设计

目标不是强行导流，而是在结果页给出自然承接。

建议承接路径：

1. 测试结果页领取“完整版解析”
2. 引导进入公众号 / 社群 / 企微
3. 通过每日内容、专题活动、节日测试做二次触达

**适合承接的免费权益：**
- 完整版 PDF 解析
- 每周运势总结
- 节日限定测试资格
- 班级/宿舍团体测试活动

### B. 轻量付费优先级

下阶段优先验证“用户是否愿意为结果深度和个性化表达付费”，不建议一开始就做复杂会员体系。

| 付费项 | 建议价格 | 目标 |
|--------|----------|------|
| 深度人格报告 | 3.9 ~ 6.9 元 | 验证结果内容付费意愿 |
| CP 深度解析 | 3.9 ~ 6.9 元 | 验证社交关系类付费 |
| 限定主题测试包 | 1.9 ~ 3.9 元 | 验证活动型内容付费 |
| 专属分享海报皮肤 | 0.9 ~ 1.9 元 | 验证轻装扮消费 |

### C. 暂缓项

以下方向可以保留，但不建议在下阶段优先投入：

- 重会员体系
- 复杂商城
- 多层级虚拟权益
- 过重的社交聊天功能

原因：当前核心矛盾仍是流量和留存，不是支付能力不足。

## 6. 传播策略设计

### 6.1 分享链路

建议形成统一传播闭环：

`测试入口 -> 结果页 -> 海报/文案分享 -> 好友进入 -> 发起同类测试/配对 -> 二次传播`

### 6.2 裂变触发点

优先做这几类“天然想晒”的结果：

- 有反差感
- 有标签感
- 有社交评价意味
- 能用于同学之间调侃
- 能作为朋友圈文案

### 6.3 文案风格

- 少讲大道理，多给“可转发的一句话”
- 少官方口吻，多像同龄人表达
- 标题要短，第一眼能懂，第二眼想发

**示例：**

- “你是那种嘴上说无所谓，心里全记住的人”
- “你在宿舍属于表面随和，实际边界感超强型”
- “你谈恋爱时最像先上头后清醒那一类”

## 7. 技术落地与后台规划

### 7.1 前端优先事项

1. 题库配置化  
2. 结果页模板化  
3. 分享海报多模板支持  
4. 节日专题入口可配置  
5. 历史记录页搭建  

### 7.2 后端基础能力

如果进入数据沉淀阶段，再增加轻量后端即可：

- 用户登录与设备标识绑定
- 测试记录存储
- 分享行为上报
- 每日内容分发
- 活动开关配置

### 7.3 后台管理规划

后台不建议一开始做成重运营平台，而是分两期建设。

#### 阶段 A：轻量后台（优先纳入 v2 阶段）

| 模块 | 功能 | 价值 |
|------|------|------|
| 题库管理 | 新增/编辑/上下线题目与结果文案 | 提高内容迭代效率 |
| 每日测试投放 | 配置日期、排序、启停状态 | 支撑持续更新 |
| 专题活动管理 | 开学季、期末季、520 等活动入口配置 | 支撑节点传播 |
| 星座内容管理 | 每日/每周文案录入与替换 | 提高回访能力 |
| 基础数据看板 | 开始率、完成率、分享率、回访率 | 支撑增长决策 |

**阶段目标：**
- 让内容和活动可配置
- 让运营侧不依赖前端发版
- 让基础增长数据可见

#### 阶段 B：正式后台（放入 v3 前后）

| 模块 | 功能 | 价值 |
|------|------|------|
| 用户管理 | 用户列表、测试记录、画像标签 | 支撑精细化运营 |
| 行为分析 | 分享来源、留存、热门测试排行 | 支撑版本优化 |
| 内容审核 | 敏感词、活动文案审核、发布流程 | 降低合规风险 |
| 引流位管理 | 公众号、社群、企微、弹窗位配置 | 支撑私域承接 |
| 商品配置 | 深度报告、测试包、海报皮肤配置 | 支撑轻量变现 |

**阶段目标：**
- 让用户和运营数据可管理
- 让引流和变现能力可配置
- 为后续商业化做基础设施准备

### 7.4 后台技术建议

| 层级 | 建议方案 |
|------|----------|
| API | Node.js + Express / NestJS（按复杂度选择） |
| 数据库 | MySQL |
| 管理端 | React + Ant Design 后台 |
| 内容配置 | JSON 结构化题库 + 表单化录入 |
| 文件存储 | 海报模板、活动素材统一管理 |

**建议原则：**
- 先做最小可用后台，不追求权限体系一步到位
- 内容配置优先，用户管理其次，商城能力后置
- 所有活动和题库尽量配置化，减少硬编码

### 7.5 风险与注意事项

- 学生用户内容要避免低俗、攻击、过度诱导
- 匿名社交类功能需慎重，优先做轻互动而非强匿名
- 星座/人格内容要强调“娱乐参考”，降低争议
- 涉及收费时需提前评估微信侧合规与支付资质

## 10. 成功判断指标

下阶段建议重点盯以下 6 个指标：

| 指标 | 判断意义 |
|------|----------|
| 测试开始率 | 首页入口是否足够吸引 |
| 测试完成率 | 题量与内容是否合理 |
| 分享触发率 | 结果是否值得发 |
| 分享带来新用户占比 | 裂变是否真实发生 |
| 7 日回访率 | 内容是否有持续吸引力 |
| 付费点击率 / 支付率 | 商业化是否值得继续做 |

---

## 11. 推荐结论

下阶段最值得投入的，不是一次性做大而全，而是优先做出三件事：

1. **补学生场景测试内容，形成更强共鸣**
2. **把结果页做成天然可传播的“社交货币”**
3. **补齐埋点和记录，为后续引流变现打基础**

如果这三件事跑顺，后面的云同步、深度报告、会员体系才有放大的意义。
